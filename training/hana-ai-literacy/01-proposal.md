## 2. 교육 배경 및 필요성

### 2.1 왜 지금인가

- 금융투자업계 전반에서 생성형 AI 활용이 급속히 확산 중. 글로벌 IB(Goldman Sachs, JPMorgan 등)는 이미 사내 LLM 도입을 공식화했으며, 국내 증권사들도 파일럿을 진행하고 있음.
- 신입사원은 디지털 친화도는 높으나 **증권사 특유의 보안·컴플라이언스 맥락**을 모른 채 공용 AI 서비스에 업무 데이터를 입력하는 리스크가 실재함.
- 사고가 난 후 교육하는 것이 아니라, 배치 전에 "안전 주행 면허"를 주는 것이 이 교육의 핵심 가치.

### 2.2 증권사 특유의 리스크

1. **MNPI(Material Non-Public Information)**: 자본시장법상 미공개중요정보를 AI에 입력하는 행위 자체가 정보 유출 경로가 될 수 있음.
2. **고객 개인정보·금융정보**: 개인정보보호법·신용정보법·금융소비자보호법 위반 리스크.
3. **이해상충·시장 질서**: 발간 전 리서치 보고서, 딜 진행 정보 등이 AI 서버로 전송될 경우 이해상충·시세조종 의심 소지.
4. **AI 환각(Hallucination)**: 시세, 재무수치, 법령 조항을 AI가 그럴듯하게 생성하지만 틀릴 수 있음. 검증 없이 고객·내부 보고에 쓰면 직접 손실 또는 법적 책임.

### 2.3 기회

- 공개 정보(뉴스·공시·번역·초안 작성 등)를 AI로 처리하면 신입사원의 반복 업무 시간을 30~70% 단축할 수 있음.
- 조기에 "올바른 AI 사용 습관"을 형성하면 개인 생산성과 회사 리스크 관리를 동시에 달성.

## 3. 교육 목표

### 3.1 핵심 목표 3가지

| # | 목표 | 측정 방법 |
|---|------|-----------|
| 1 | **보안 판단력**: 어떤 데이터를 AI에 넣으면 안 되는지 3색 신호등 기준으로 즉시 판단할 수 있다 | 교육 후 신호등 퀴즈 20문항 정답률 ≥ 90% |
| 2 | **워크플로 내재화**: 자신의 배치 부서와 연관된 AI 활용 시나리오 1개 이상을 스스로 설명하고 실행할 수 있다 | 핸즈온 결과물 발표 (자기평가 + 동료평가) |
| 3 | **프롬프팅 기본기**: 좋은 프롬프트와 나쁜 프롬프트의 차이를 알고 업무용 프롬프트를 작성할 수 있다 | Before/After 프롬프트 개선 과제 |

### 3.2 부수 목표

- AI를 과도하게 두려워하거나, 반대로 맹신하는 양극단을 탈피해 **도구적 실용 관점** 정착
- AI 관련 신기술·정책 변화를 스스로 팔로우하는 학습 습관 형성

## 4. 교육 대상 분석

### 4.1 공통 특성

- 대졸 신입 (22~27세 내외), 디지털 서비스 소비 경험은 풍부
- ChatGPT 등 공용 AI 서비스 개인 경험 보유 가능성 높으나, **업무 맥락에서의 리스크 인식은 낮음**
- 전공: 경영·경제·수학·통계·컴공 등 다양, 코딩 수준 혼재

### 4.2 배치 부서별 관심 포인트

| 부서 | 주요 업무 | AI 관심 포인트 |
|------|-----------|----------------|
| WM/PB | 고객 자산관리, 상품 제안 | 고객 소통 자료, 시황 요약 |
| IB | 딜 구조화, IM 작성, 실사 | 문서 초안, 번역, 리서치 정리 |
| S&T | 시장 트레이딩, 리스크 관리 | 뉴스 요약, 코드 보조 |
| 리서치 | 기업분석, 보고서 작성 | 공시 요약, 번역, 차트 보조 |
| 리테일/지점 | 고객 응대, 영업 지원 | 안내문, FAQ, 세미나 자료 |
| 백오피스/IT/컴플 | 지원·운영·준법 | 엑셀/코드 보조, 문서 정리 |

## 5. 교육 설계 원칙

### 5.1 "3색 신호등" 프레임 (전 모듈 공통)

교육 전체를 관통하는 단일 판단 기준:

```
🔴 빨강 — 절대 금지 (AI 입력 불가, 위반 시 징계·법적 책임)
🟡 노랑 — 사내 승인 도구로만 (공용 AI 사용 금지, 사내 시스템 내에서만)
🟢 초록 — 공용 AI 사용 가능 (단, 출력 결과 반드시 검증 후 사용)
```

### 5.2 "금지"가 아닌 "안전한 활용" 중심

- 보안 교육이지만 "전부 금지" 메시지가 아닌, **올바른 사용 경로를 명확히 알려주는** 방식
- 각 금지 케이스마다 "대신 이렇게 하세요"를 반드시 제시

### 5.3 실전 중심, 이론 최소화

- AI 기술 설명은 업무 활용에 필요한 최소 수준으로 제한
- 부서별 실제 발생 가능한 시나리오와 프롬프트 예시를 최대한 구체적으로 제공

### 5.4 검증 습관화

- AI 출력을 그대로 쓰지 않고 **검증 체크리스트 3단계**를 항상 거치도록 습관화
  1. 수치/사실 확인 (출처 대조)
  2. 컴플라이언스 자가 체크 (3색 신호등)
  3. 상급자 리뷰 전달 전 최종 감수

## 6. 교육 운영 계획

### 6.1 강의 구성 요약

> **기본 단위**: 50분 강의 + 10분 휴식 × 5교시 = 총 300분 (5시간)

| 교시 | 시간 | 내용 | 포함 모듈 |
|------|------|------|-----------|
| **1교시** | 50분 | 도입 + 증권사 보안 인트로 | M1 전체 + M2 전반 |
| — | 10분 | 휴식 | |
| **2교시** | 50분 | 증권사 보안 심화 + 신호등 퀴즈 | M2 후반 |
| — | 10분 | 휴식 | |
| **3교시** | 50분 | AI 원리 + 프롬프팅 기본기 | M3 전체 |
| — | 10분 | 휴식 | |
| **4교시** | 50분 | 부서별 활용 시나리오 | M4 전체 |
| — | 10분 | 휴식 | |
| **5교시** | 50분 | 핸즈온 + 마무리·Q&A | M5 + M6 |
| **합계** | **300분** | | |

### 6.2 강의 운영 형식

- **강의**: M1·M2·M3 — 슬라이드 중심, 강사 설명 + 퀴즈/질문 삽입
- **워크숍**: M4 — 부서별 소그룹(4~6명) 토론 후 전체 공유 (게스트 스피커 참여 가능)
- **핸즈온**: M5 — 트랙 A(노트북 실습) 또는 트랙 B(종이 워크시트 + 강사 시연) 중 환경에 맞게 선택
- **Q&A**: M6 — 사전 익명 질문 수집(슬리도/QR 폼 활용) + 현장 질의

### 6.3 준비물 및 사전 체크리스트

**강사 준비**
- [ ] 슬라이드 완성본 (PPT/Keynote)
- [ ] 보안·컴플 퀴즈 20문항 배포 자료 (종이 또는 온라인)
- [ ] 부서별 플레이북 인쇄본 (1인 1부)
- [ ] 보안 체크리스트 인쇄본 (1인 1부, 책상 비치용)
- [ ] M5 트랙 A용 실습 가이드 시트 또는 트랙 B용 종이 워크시트

**수강생 준비**
- [ ] (트랙 A 해당 시) 노트북 지참 + 사내 승인 AI 도구 또는 ChatGPT 개인 계정 로그인
- [ ] 사전 설문: "현재 AI 사용 빈도·목적" (선택 사항)

### 6.4 평가 및 피드백

| 평가 항목 | 비중 | 방법 |
|-----------|------|------|
| 보안 퀴즈 (M2 직후) | 50% | 20문항 객관식, 정답률 기록 |
| 핸즈온 결과물 (M5) | 30% | 자기평가 + 동료평가 루브릭 |
| 교육 만족도 조사 | 20% | 5점 척도 + 자유 기술 |

- 퀴즈 정답률 80% 미만 수강생 → 보안 모듈 보충 자료 개별 발송
- 교육 후 30일·90일 시점 짧은 팔로업 설문(5문항) 실시해 현업 적용 여부 추적

## 8. 향후 발전 방향

1. **심화 과정**: 부서별 맞춤형 AI 워크숍 (2~3시간, 배치 3개월 후)
2. **사내 AI 가이드라인 문서화**: 본 교육 내용을 사내 AI 사용 정책으로 공식화
3. **갱신 주기**: AI 도구 및 법령 변화에 맞춰 연 1회 이상 커리큘럼 업데이트
4. **강사 양성**: 본 교육 수료자 중 희망자를 차기 기수 보조 강사로 육성
