# 지식베이스 가이드

지식베이스는 Alli Works가 답변을 생성할 때 참조하는 문서와 Q&A 데이터를 관리하는 핵심 기능입니다.

---

## 1. 문서 관리

### 문서 업로드
**경로**: Alli 대시보드 > 지식베이스 > 문서

#### 지원 파일 형식
`.txt`, `.docx`, `.csv`, `.xls`, `.xlsx`, `.xlsm`, `.pdf`, `.ppt`, `.pptx`, `.doc`

#### 업로드 방법
1. 폴더 생성: 주제별/부서별 폴더 구조 생성
2. 파일 업로드: 드래그 앤 드롭 또는 파일 선택
3. 해시태그 부여: 문서별 검색 분류 태그
4. 접근 권한 설정: 폴더별 접근 가능 그룹/멤버 지정

#### 폴더 구조 예시
```
├── 고객지원/
│   ├── 상품설명서/ (#ELS, #DLS, #펀드)
│   ├── 수수료/ (#수수료, #해외거래)
│   └── 계좌안내/ (#계좌개설, #비대면)
├── 자격증/
│   ├── 투자자문인력/ (#투자자문)
│   ├── 펀드투자상담사/ (#펀드상담사)
│   └── 파생상품투자상담사/ (#파생상품)
├── SOP/
│   ├── 주문체결/ (#주문, #체결)
│   ├── 정산/ (#정산)
│   └── 리스크/ (#리스크, #마진)
└── 규정/
    ├── 자본시장법/ (#불공정거래)
    └── 내부통제/ (#정보장벽)
```

### 해시태그 일괄 관리
- 대시보드 > 지식베이스 > 문서 > 해시태그 관리
- 여러 문서에 동시에 해시태그 부여/제거 가능
- 답변 생성 노드에서 특정 해시태그 문서만 검색 범위로 지정 가능

### 엑셀 테이블 데이터 활용
- .xlsx 파일을 테이블 데이터로 업로드하면 행/열 단위로 검색 가능
- 수치 데이터가 포함된 문서에 효과적

## 2. Q&A 관리

### Q&A 등록 방법

#### 방법 1: 개별 등록
**경로**: 지식베이스 > Q&A > "+ 새 Q&A"
- 질문과 답변을 직접 입력
- 해시태그, 카테고리 지정

#### 방법 2: CSV 일괄 업로드
**경로**: 지식베이스 > Q&A > 일괄 업로드

CSV 형식:
```csv
질문,답변
"주식 매매 수수료는 얼마인가요?","온라인 거래 시 0.015%, 오프라인 거래 시 0.1%입니다."
"계좌 개설은 어떻게 하나요?","모바일 앱에서 비대면 개설이 가능합니다."
```

#### 방법 3: 복수 질문 등록
- 하나의 답변에 여러 질문 변형을 등록
- 다양한 표현으로 같은 의미의 질문을 매칭

#### 방법 4: 문서에서 Q&A 자동 생성
**경로**: 지식베이스 > Q&A > 문서에서 Q&A 자동 생성
- 업로드된 문서 내용을 AI가 분석하여 질문-답변 쌍 자동 생성
- 생성 후 검토/수정/삭제 가능

### Q&A 설정
- 매칭 정확도 임계값 조정
- 해시태그 기반 검색 범위 제한
- 우선순위 설정

## 3. 동의어/이의어 사전

**경로**: Alli 대시보드 > 학습 > 동의어 사전

사용자가 다양한 용어로 검색해도 동일한 결과를 얻을 수 있도록 동의어를 등록합니다.

### 등록 방법
개별 등록 또는 CSV 일괄 업로드:

```csv
대표어,동의어1,동의어2,동의어3,동의어4
해외주식,외국주식,미국주식,글로벌주식,
환전,외화매매,환율적용,,
출금,이체,송금,자금이체,
수수료,거래비용,매매수수료,위탁수수료,
계좌개설,계좌만들기,계좌신규,,
```

### 효과
- "해외주식"으로 검색해도, "외국주식", "미국주식" 등의 결과가 함께 표시
- 검색 정확도와 재현율 동시 향상

## 4. 리트리버 최적화

**경로**: 학습 > 리트리버 최적화

RAG 검색 성능을 튜닝합니다:
- 검색 알고리즘 파라미터 조정
- 청크 크기/오버랩 설정
- 리랭킹 모델 활성화

---

## 5. 피드백 관리

**경로**: 학습 > 피드백

### 피드백 유형
- **긍정 (👍)**: 정확하고 유용한 답변
- **부정 (👎)**: 부정확하거나 부적절한 답변
  - 올바른 정보 입력 가능
  - 문제 유형 선택: 질문과 무관 / 정보 부정확 / 더 나은 답변 기대

### 피드백 활용
- 부정 피드백이 많은 질문을 분석하여 Q&A 추가/수정
- 문서 내용 업데이트 필요 여부 판단
- 프롬프트 개선 방향 도출
