# 대화형 앱 노드 가이드

대화형 앱은 다양한 노드를 연결하여 대화 흐름을 설계합니다. 각 노드의 설정 방법을 상세히 설명합니다.

---

## 노드 목록

| 노드명 | 영문 | 용도 |
|--------|------|------|
| **메시지 보내기** | Send Message | 사용자에게 메시지 표시 / 질문하기 |
| **답변 생성** | Answer Generation | 문서/Q&A 기반 RAG 답변 생성 |
| **Q&A에서 답변** | Answer in Q&A | Q&A 지식베이스에서 직접 매칭 |
| **조건 추가** | Add Condition | 변수값 기반 조건 분기 |
| **담당 멤버 연결** | Assign Member Connection | 실시간 담당자 채팅 연결 |
| **입력 폼 보내기** | Send Input Form | 구조화된 입력 양식 표시 |
| **LLM 인풋** | LLM Input | LLM 처리를 위한 입력 소스 설정 |
| **LLM 노드** | LLM Node | 프롬프트 기반 LLM 처리 |
| **딥 리서치** | Deep Research | 문서+인터넷 심층 연구 |
| **변수값 설정** | Set Variable | 변수에 값 할당 / 계산식 |
| **이메일 보내기** | Send Email | 이메일 자동 발송 |
| **연동하기** | Integration | 외부 시스템 연동 (API 호출) |
| **MCP 노드** | MCP | Model Context Protocol 도구 호출 |

## 1. 메시지 보내기 노드

사용자에게 텍스트 메시지를 표시하거나 질문을 던지는 노드입니다.

### 설정 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **메시지 내용** | 사용자에게 표시할 텍스트 | "안녕하세요, 인재개발실 입니다." |
| **메시지 유형** | 텍스트 / 리치 텍스트 / 이미지 | 텍스트 |
| **질문하기 옵션** | 사용자 입력을 받을지 여부 | 텍스트 입력 / 버튼 선택 |
| **변수 저장** | 사용자 응답을 변수에 저장 | @USER_INPUT |
| **다음 노드** | 자동 진행 여부 | 예 |

### 활용 팁
- 인사말, 안내 메시지에 사용
- 질문하기 옵션으로 사용자 입력을 받아 변수에 저장 가능
- 버튼 선택형 질문으로 사용자 의도를 명확하게 파악

- [메시지 보내기/질문하기 노드](https://docs.allganize.ai/allganize-alli-works/alli-dashboard/app-management/conversation-app/nodes/undefined-2)

## 2. 답변 생성 노드

업로드된 문서와 Q&A를 검색하여 AI가 답변을 생성하는 핵심 노드입니다.

### 설정 항목

| 항목 | 설명 | 권장값 |
|------|------|--------|
| **질문 입력 방식** | 멤버 입력 / 변수 | 멤버 입력 (사용자 직접 질문) |
| **노드 진입 시 메시지 출력** | 안내 메시지 표시 여부 | ☑ 체크 |
| **기본 모델** | LLM 모델 선택 | AZURE GPT-4o |
| **에이전트** | RAG 에이전트 선택 | 기본 RAG (기본) |
| **검색 소스 - Q&A** | Q&A 지식베이스 검색 | 필요시 ☑ 체크 |
| **검색 소스 - 문서** | 문서 지식베이스 검색 | ☑ 체크 → 폴더 지정 |
| **검색 소스 - 인터넷** | 웹 검색 | 필요시 ☑ 체크 |
| **답변 변수 저장** | 답변을 변수에 저장 | @TEXT 또는 @ANSWER |
| **답변 생성 후** | 다음 노드 진행 / 다시 질문 | 다음 노드로 진행 |
| **분기 옵션** | 정답 미발견 시 분기 | 선택사항 |

### 에이전트 편집
"편집" 버튼을 클릭하면 에이전트의 시스템 프롬프트를 커스터마이징할 수 있습니다.

```
시스템 프롬프트 예시:
"당신은 하나증권의 [역할] 전문가입니다. 
업로드된 문서를 기반으로 정확한 정보를 제공합니다.
문서에 없는 내용은 '확인이 필요합니다'라고 안내합니다."
```

### 검색 소스 설정 방법
1. **Q&A**: 체크 시 등록된 Q&A에서 직접 매칭 (정확도 높음)
2. **문서**: 체크 후 펼쳐서 특정 폴더를 지정 (범위 제한)
3. **인터넷**: 체크 시 웹 검색 결과도 참조 (최신 정보 필요 시)

### 참고 링크
- [답변 생성 노드](https://docs.allganize.ai/allganize-alli-works/alli-dashboard/app-management/conversation-app/nodes/generativeanswer)

## 3. Q&A에서 답변 노드

Q&A 지식베이스에 등록된 질문-답변 쌍에서 직접 매칭하여 답변합니다.

### 설정 항목
| 항목 | 설명 |
|------|------|
| **Q&A 소스** | 검색할 Q&A 카테고리/해시태그 지정 |
| **매칭 정확도** | 유사도 임계값 설정 |
| **미매칭 시** | 다른 노드로 분기 가능 |

### 답변 생성 노드와의 차이
- Q&A에서 답변: 등록된 답변을 그대로 반환 (정확도 최고)
- 답변 생성: LLM이 문서를 참조하여 새로운 답변 생성 (유연성 높음)

## 4. 조건 추가 노드

변수값을 기반으로 대화 흐름을 분기합니다.

### 설정 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **옵션 (조건)** | 변수 + 연산자 + 값 | @TEXT contains "없습니다" |
| **+ 옵션 추가** | 추가 조건 분기 | @TEXT contains "찾을 수 없" |
| **기타 (else)** | 조건에 해당하지 않을 때 | 정상 답변 처리 |
| **메모** | 조건 분기 로직 설명 | 자유 입력 |

### 사용 가능한 연산자

| 연산자 | 설명 | 예시 |
|--------|------|------|
| `contains` | 특정 문자열 포함 | @TEXT contains "없습니다" |
| `equals` | 정확히 일치 | @TYPE equals "주식" |
| `not contains` | 특정 문자열 미포함 | @TEXT not contains "성공" |
| `is empty` | 변수가 비어있음 | @ANSWER is empty |
| `is not empty` | 변수에 값이 있음 | @ANSWER is not empty |
| `starts with` | 특정 문자열로 시작 | @TEXT starts with "오류" |

### 활용 패턴
1. **답변 실패 감지**: @ANSWER contains "없습니다" → 담당자 연결
2. **위험도 분기**: @분석결과 contains "높음" → 에스컬레이션
3. **사용자 선택 분기**: @USER_CHOICE equals "상품문의" → 상품 노드

### 참고 링크
- [조건 추가 노드](https://docs.allganize.ai/allganize-alli-works/alli-dashboard/app-management/conversation-app/nodes/condition)

## 5. 담당 멤버 연결 노드

AI가 처리하지 못한 문의를 실시간으로 담당자에게 연결합니다.

### 설정 항목

| 항목 | 설명 |
|------|------|
| **연결 방식** | 내부 메시지 발송 후 담당 멤버와 채팅 연결 |
| **담당 멤버/팀** | 특정 멤버 또는 팀(순차 배정) 선택 |
| **대기 메시지** | 연결 중 사용자에게 표시할 메시지 |
| **이메일 알림** | 담당자에게 이메일 알림 발송 여부 |
| **대화 내역 전달** | 이전 대화 컨텍스트를 담당자에게 전달 |

### 활용 팁
- 조건 추가 노드와 함께 사용하여 에스컬레이션 구성
- 업무시간 외 대기 메시지에 안내 추가 권장
- 대화 내역 전달을 활성화하면 담당자가 맥락을 바로 파악 가능

### 참고 링크
- [담당 멤버 연결 노드](https://docs.allganize.ai/allganize-alli-works/alli-dashboard/app-management/conversation-app/nodes/contact)

## 9. 딥 리서치 노드

문서와 인터넷을 활용한 심층 연구 보고서를 자동 생성합니다.

### 주의사항
- **반드시 마지막 노드(터미널 노드)로 배치해야 합니다**
- 비동기 실행: 10~30분 소요 가능
- 완료 시 출처가 포함된 연구 보고서 생성

### 검색 소스 설정
1. **문서**: 업로드된 폴더 지정
2. **인터넷**: 신뢰 URL 지정 가능
3. **업로드 파일**: 사용자가 추가 파일 첨부

## 노드 연결 패턴 (워크플로우 예시)

### 패턴 1: 기본 FAQ 챗봇
```
시작 → 메시지 보내기 (인사말) → 답변 생성 (문서+Q&A) → 조건 추가 → 담당 멤버 연결
```

### 패턴 2: 딥 리서치
```
시작 → 메시지 보내기 (주제 입력) → 딥 리서치 [터미널]
```

### 패턴 3: 컴플라이언스 점검
```
시작 → 입력 폼 보내기 → 답변 생성 → LLM 노드 → 조건 추가 → 담당 멤버 연결 / 메시지 보내기
```

### 패턴 4: 리서치 요약
```
시작 → LLM 인풋 (문서 선택) → LLM 노드 (요약 프롬프트) → 메시지 보내기 (결과)
```
