## 세션 1. Alli Works 소개 & 왜 Azure인가? (00:00 ~ 00:25, 25분)

**목표**: "이 도구가 뭔지, 왜 Azure를 쓰는지" 이해

### 1-1. Alli Works란? (10분)

Allganize의 기업용 AI 플랫폼입니다. 핵심 가치는 사내 문서를 AI에 연결해서 챗봇/에이전트/답변 앱을 **코딩 없이** 만드는 것입니다.

**3가지 앱 타입** (오늘은 "대화형 앱"에 집중)

| 앱 타입 | 특징 | 오늘 실습 |
|---------|------|----------|
| 대화형 앱 | 노드를 드래그&드롭으로 연결하는 플로우 빌더 | **O (메인)** |
| 에이전트형 앱 | 자연어 인스트럭션 + MCP 도구 | 데모만 |
| 답변형 앱 | 단일 입출력 (보고서 요약 등) | 소개만 |

### 1-2. 왜 Azure OpenAI인가? (8분)

> 강사 멘트: "GPT-4o 모델 자체는 OpenAI 직접 쓰든 Azure로 쓰든 **동일한 성능**입니다"

**Azure를 써야 하는 이유 3가지:**

1. **데이터 보안** — 고객/직원 데이터가 OpenAI 모델 학습에 사용되지 않음 (엔터프라이즈 계약)
2. **금융보안원 규정** — 금융사 클라우드 이용 가이드라인 충족, Korea Central 리전 활용
3. **감사 추적** — 엔터프라이즈급 로깅/모니터링으로 규제 대응 가능

**Azure vs OpenAI 비교표**

| 기능 | Azure OpenAI | OpenAI 직접 |
|------|-------------|------------|
| GPT-4o 모델 성능 | 동일 | 동일 |
| 인터넷 검색 | O | O |
| 문서 RAG | O | O |
| 딥 리서치 | O | O |
| 에이전트/MCP | O | O |
| 고객 데이터 학습 | **사용 안 함** | 기본 사용 가능 |
| 한국 리전 | Korea Central | 불가 |
| 최신 모델 출시 | 1~3개월 뒤 | 즉시 |

> 결론: 기능/성능 차이 없음. 보안과 규정 때문에 Azure가 필수.

### 1-3. 오늘 만들 것 미리보기 (7분)

완성된 "자격증 Q&A 챗봇" 라이브 데모를 보여줍니다.

- **정상 답변**: "투자자문인력 시험 일정은?" → AI 자동 답변
- **실패 → 담당자 연결**: "파생상품투자상담사 환불 규정" → AI 실패 → 담당자에게 자동 연결

**노드 구성도 (4개 노드)**

```
Start
  ↓
메시지 보내기 — "안녕하세요, 인재개발실입니다."
  ↓
답변 생성 — AZURE GPT-4o / 문서 검색 / 변수 @TEXT 저장
  ↓
조건 추가 — @TEXT contains "없습니다"?
  ├─ Yes → 담당 멤버 연결 (인재개발실)
  └─ No  → 답변 성공 (정상 전달)
```

## 세션 3. 지식베이스 실습 (00:55 ~ 01:20, 25분)

**목표**: 챗봇이 참조할 데이터를 직접 올려보기

### 3-1. Q&A 일괄 업로드 (10분)

**메뉴**: 지식베이스 > Q&A > 일괄 업로드

1. 제공된 `cert-qa-sample.csv` 파일 업로드 (30건)
2. CSV 형식 확인:
   ```
   질문,답변
   "투자자문인력 자격시험 응시 요건은?","학력/경력 제한 없이 누구나 응시 가능합니다..."
   "펀드투자상담사 시험과목은?","1과목: 펀드투자, 2과목: 증권투자, 3과목: 부동산투자..."
   ```
3. 업로드 후 확인: Q&A 목록에 30건 등록 확인
4. 해시태그 부여 실습:
   - `#투자자문인력`
   - `#펀드투자상담사`
   - `#파생상품투자상담사`
   - `#시험일정`
   - `#갱신절차`

### 3-2. 문서 업로드 (15분)

**메뉴**: 지식베이스 > 문서 > 새 폴더 > 업로드

폴더 구조를 함께 만듭니다:

```

├── 투자자문인력/        (시험안내.pdf)
├── 펀드투자상담사/      (학습가이드.txt)
└── 파생상품투자상담사/  (시험일정.md)
```

- 각 문서에 해시태그 부여 (#투자자문인력 등)
- 접근 권한: "전체 직원"으로 설정

**체크포인트**: 문서 상태가 "처리 완료"인지 반드시 확인 (AI가 읽을 수 있는 상태)

## 세션 5. 조건 분기 & 담당자 연결 실습 (02:10 ~ 02:35, 25분)

**목표**: AI가 답변 실패하면 사람에게 자동 연결하는 안전장치 구축

### 5-1. 조건 추가 노드 (10분)

답변 생성 노드 출력에서 드래그 → **조건 추가 노드** 추가

**조건 설정:**

```
옵션 1:
  변수:   @TEXT
  연산자: contains
  값:     "없습니다"
  → 참이면: "담당 멤버 연결" 노드로 이동

기타 (else):
  → 정상 종료
```

> 강사 설명: "답변 생성 노드가 문서에서 답을 못 찾으면 '관련 정보를 찾지 못했습니다'라고 응답합니다. 그 문장에 '없습니다'가 포함돼 있으니, 이걸 감지해서 사람한테 넘기는 겁니다."

### 5-2. 담당 멤버 연결 노드 (10분)

조건 추가 노드의 "옵션 1" 출력에서 드래그 → **담당 멤버 연결 노드** 추가

| 설정 항목 | 설정값 |
|-----------|--------|
| 연결 방식 | 내부 메시지 발송 후 담당 멤버와 채팅 연결 |
| 담당 멤버 | **교육생 2인 1조로 서로 지정** (실습) |
| 대기 메시지 | "담당자 연결 중입니다. 잠시만 기다려주세요." |
| 이메일 알림 | 활성화 |
| 대화 내역 전달 | 활성화 |

### 5-3. 담당자 역할 체험 — 2인 1조 실습 (5분)

**A역할 (질문하는 직원)**:
- 챗봇에 "파생상품투자상담사 환불 규정 알려줘" 질문 (의도적 실패 유도)

**B역할 (담당자)**:
1. Alli 대시보드 > **대화** > **대기 중** 탭에서 알림 확인
2. 해당 대화 클릭 → AI가 나눈 이전 대화 전체 확인
3. **[수락]** 버튼 클릭
4. A와 실시간 채팅 시작

역할 교대하여 반복합니다.

> 강사 멘트: "담당자는 별도 앱 설치 없이 웹 브라우저 하나로 수락하고 채팅할 수 있습니다. 야간/주말에는 자동으로 '연결 실패' 메시지가 나갑니다."

**담당자 연결 전체 흐름 정리:**

```
직원이 질문 → AI 답변 시도 → 실패 감지 (@TEXT contains "없습니다")
  ↓
"담당 멤버 연결" 노드 실행
  ↓
담당자에게 알림 (브라우저 알림 + 이메일)
  ├─ 알림에 "이전 대화 요약" 포함
  ↓
담당자: 대시보드 > 대화 > 대기 중 > [수락]
  ↓
직원 ↔ 담당자 실시간 채팅 시작
```

**업무 시간 외 처리:**
- 담당자 미수락 시 → "연결 실패 메시지" 자동 발송
- 예: "현재 업무시간(09:00~18:00)이 아닙니다. 내일 인재개발실(내선 XXXX)로 연락 바랍니다."

### 6-1. 종합 테스트 (8분)

| 4 | "파생상품투자상담사 환불 규정" | 답변 실패 → 담당자 연결 |

### 6-2. 피드백 학습 (7분)

1. 부정확한 답변에 👎 (thumbs down) 피드백 클릭
2. 올바른 정보를 직접 입력
3. 같은 질문을 다시 해서 개선 확인
