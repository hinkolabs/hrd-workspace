# 에이전트형 앱 가이드

에이전트형 앱은 LLM에 인스트럭션과 도구를 제공하여 자율적으로 작업을 수행하는 AI 에이전트입니다.

---

## 1. 에이전트 생성

**경로**: Alli 대시보드 > 앱 관리 > + 새 앱 만들기 > 에이전트 만들기

### 기본 정보 설정
| 항목 | 설명 | 예시 |
|------|------|------|
| **앱 이름** | 에이전트 표시 이름 | "WM 상담 도우미" |
| **아이콘** | 앱 아이콘 | 이모지 또는 이미지 |
| **설명** | 에이전트 역할 설명 | "고객 투자 성향에 맞는 금융상품을 분석합니다" |
| **카테고리** | 앱 분류 | WM / IB / S&T / 경영지원 |

---

## 2. LLM 모델 및 인스트럭션

### 모델 선택
| 모델 | 특징 | 권장 용도 |
|------|------|-----------|
| **AZURE GPT-4o** | 높은 정확도, 멀티모달 지원 | 복잡한 분석, 문서 이해 |
| **GPT-4o-mini** | 빠른 응답, 낮은 비용 | 간단한 질의응답 |
| **Claude** | 긴 컨텍스트, 안전성 | 장문 문서 분석 |

### 인스트럭션 작성

인스트럭션은 에이전트의 역할, 행동 규칙, 제약사항을 정의하는 시스템 프롬프트입니다.

#### 인스트럭션 템플릿
```
[역할 정의]
당신은 하나증권 {부서}의 {역할} 전문가입니다.

[행동 규칙]
1. 업로드된 문서를 기반으로 정확한 정보를 제공합니다.
2. 수치/데이터는 반드시 출처를 명시합니다.
3. 비교 분석 시 표 형태로 정리합니다.

[제약사항]
- 문서에 없는 내용은 "확인이 필요합니다"라고 안내합니다.
- 투자 추천/자문은 절대 하지 않습니다.
- 개인정보는 포함하지 않습니다.

[출력 형식]
Markdown 형식으로 구조화하여 응답합니다.
```

#### 인스트럭션 예시 (WM 상담)
```
하나증권 WM 상담 전문가로서 고객 투자 성향({customer_risk_profile})에 맞는 
상품을 분석합니다. 수익률/리스크/수수료를 비교표로 정리합니다.
모든 분석은 업로드된 약관과 투자설명서를 기반으로 합니다.
```

---

## 3. 사용자 변수

에이전트 인스트럭션에서 `{변수명}` 형태로 동적 입력을 처리합니다.

### 변수 설정 방법
1. 인스트럭션에서 `{변수명}` 형태로 참조
2. 변수 탭에서 변수 유형 및 설명 설정
3. 사용자가 에이전트 실행 시 변수값 입력

### 예시
| 변수명 | 유형 | 설명 |
|--------|------|------|
| `{customer_risk_profile}` | 선택형 | 안정형/안정추구형/위험중립형/적극투자형/공격투자형 |
| `{target_product}` | 텍스트 | 분석 대상 상품명 |
| `{analysis_period}` | 텍스트 | 분석 기간 (예: 최근 6개월) |

## 4. MCP 도구

### MCP (Model Context Protocol) 개요
에이전트가 외부 도구를 호출하여 작업을 수행할 수 있게 합니다.

### 기본 제공 도구

| 도구 | 설명 |
|------|------|
| **문서 검색** | 업로드된 문서에서 관련 내용 검색 |
| **웹 검색** | 인터넷에서 최신 정보 검색 |
| **코드 실행** | Python 코드 실행 (데이터 분석) |

### 도구 추가 방법
1. 에이전트 편집 > 도구 탭
2. Alli MCP Hub에서 사전 연동된 도구 선택
3. 각 도구의 인증/인자 설정
4. 도구 사용 조건 및 설명 작성

## 5. 대화 시나리오

### 시작 메시지
에이전트 첫 접속 시 표시할 메시지:
```
"안녕하세요! WM 상담 도우미입니다. 
고객 투자 성향에 맞는 금융상품을 분석해 드리겠습니다. 
궁금하신 점을 말씀해 주세요."
```

### 실패 메시지
답변 불가 시 표시할 메시지:
```
"죄송합니다. 해당 내용은 현재 등록된 문서에서 확인되지 않습니다. 
담당자에게 문의해 주세요."
```

---

## 6. 버전 관리

에이전트는 버전 관리를 통해 반복적으로 개선할 수 있습니다.

### 워크플로우
1. 인스트럭션/도구 설정 후 **v1 저장**
2. 미리보기로 테스트
3. 인스트럭션 수정 → **v2 저장**
4. 이전 버전과 비교 테스트
5. 최종 버전 **"공개하기"**

### 버전 비교
- 각 버전의 인스트럭션 차이 확인
- 동일 질의에 대한 응답 품질 비교
- 필요시 이전 버전으로 롤백

---

## 7. Agent 워크플로우 구축

고급 사용자를 위한 에이전트 워크플로우 기능:
- 여러 에이전트를 체이닝하여 복잡한 작업 자동화
- 에이전트 간 결과 전달
- 조건부 실행 경로 설정
