# 시작하기 (관리자 가이드)

## 1. Alli 대시보드 접속

Alli 대시보드는 Alli Works의 모든 설정과 관리 기능을 제공하는 관리자 화면입니다.

### 접속 방법
1. 관리자 계정으로 Alli 대시보드에 로그인
2. 프로젝트를 선택하거나 새 프로젝트 생성
3. 좌측 메뉴에서 원하는 관리 기능으로 이동

### 메인 메뉴 구조

| 메뉴 | 설명 |
|------|------|
| **앱 관리** | 대화형/에이전트형/답변형 앱 생성 및 관리 |
| **지식베이스** | 문서 업로드, Q&A 관리, 연동 설정 |
| **학습** | 동의어 사전, 리트리버 최적화, 피드백 관리 |
| **사용 기록** | 앱별 사용 이력, 멤버 정보, LLM 통계 |
| **프롬프트 관리** | 시스템/앱 프롬프트 중앙 관리 |
| **설정** | 일반, 멤버/그룹, 보안, 변수, Alli Works 설정 |

---

## 2. 처음 사용하기 가이드

### Step 1: 프로젝트 생성
- 대시보드 좌측 상단에서 프로젝트 이름 클릭 → "새 프로젝트 만들기"
- 프로젝트 이름, 설명, 기본 언어를 설정

### Step 2: 지식베이스 구축
1. **문서 업로드**: 지식베이스 > 문서 > 폴더 생성 후 PDF/DOCX/TXT 등 업로드
2. **Q&A 등록**: 지식베이스 > Q&A > 개별 등록 또는 CSV 일괄 업로드
3. **해시태그 분류**: 문서/Q&A에 해시태그를 부여하여 검색 범위 제어

### Step 3: 앱 생성
1. 앱 관리 > "+ 새 앱 만들기"
2. 앱 유형 선택 (대화형/에이전트형/답변형)
3. 노드 구성 또는 인스트럭션 작성
4. 미리보기로 테스트 후 공개

### Step 4: 사용자 배포
1. 설정 > 멤버 및 그룹에서 사용자 초대
2. 앱별 접근 권한 설정
3. Alli Works URL 또는 커스텀 도메인 공유

## 3. 주요 개념

### 앱 유형별 특징

| 앱 유형 | 설명 | 사용 사례 |
|---------|------|-----------|
| **대화형 앱** | 노드 기반 워크플로우로 대화 흐름을 설계 | FAQ 챗봇, 조건부 안내, 업무 프로세스 |
| **에이전트형 앱** | LLM + 도구(MCP)로 자율적 작업 수행 | 문서 검색 에이전트, 업무 자동화 |
| **답변형 앱** | 단일 입력 → 단일 출력 구조 | 보고서 생성, 번역, 요약 |

### RAG (Retrieval-Augmented Generation)
Alli는 RAG 기술을 활용하여 업로드된 문서/Q&A에서 관련 정보를 검색한 후, 이를 바탕으로 LLM이 답변을 생성합니다. 이를 통해 환각(hallucination)을 최소화하고 출처가 명확한 답변을 제공합니다.

### 변수 시스템
노드 간 데이터를 전달하기 위해 변수(@변수명)를 사용합니다. 예: @TEXT, @ANSWER 등. 답변 생성 노드의 출력을 변수에 저장하고, 조건 추가 노드에서 해당 변수를 참조하여 분기 처리합니다.
