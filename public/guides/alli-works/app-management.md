# 앱 관리 개요

Alli Works에서는 3가지 유형의 앱을 생성할 수 있습니다. 각 앱 유형은 서로 다른 사용 목적과 구성 방식을 가집니다.

---

## 1. 대화형 앱 (Interactive App)

### 개요
노드 기반 플로우 빌더로 대화 흐름을 시각적으로 설계합니다. 복잡한 조건 분기, 다단계 프로세스, 외부 연동이 필요한 경우에 적합합니다.

### 주요 특징
- **시각적 플로우 빌더**: 드래그 앤 드롭으로 노드 연결
- **다양한 노드 타입**: 메시지 보내기, 답변 생성, 조건 추가, LLM 노드 등 14종+
- **변수 시스템**: 노드 간 데이터 전달 (@TEXT, @ANSWER 등)
- **조건 분기**: 변수값에 따라 다른 경로로 흐름 제어

### 사용 사례
- 고객 FAQ 자동 응대 챗봇
- 자격증 Q&A 챗봇
- 컴플라이언스 규정 점검
- 딥 리서치 자동화

### 생성 방법
1. 앱 관리 > "+ 새 앱 만들기" > "대화형 앱" 선택
2. 앱 이름/설명 입력
3. 플로우 빌더에서 노드 추가 및 연결
4. 미리보기로 테스트 → 공개

> 상세 노드 설정은 [대화형 앱 노드 가이드](conversation-app-nodes.md)를 참고하세요.

## 2. 에이전트형 앱 (Agent App)

### 개요
LLM에 인스트럭션(시스템 프롬프트)과 도구(MCP)를 제공하여 자율적으로 작업을 수행하는 앱입니다. 노드 대신 자연어 인스트럭션으로 동작을 정의합니다.

### 주요 특징
- **인스트럭션 기반**: 시스템 프롬프트로 에이전트 역할/행동 정의
- **MCP 도구 연동**: 문서 검색, 웹 검색, API 호출 등 도구 사용
- **사용자 변수**: {변수명} 형태로 동적 입력 처리
- **버전 관리**: v1 → v2 → ... 반복 개선 후 선택 공개

### 사용 사례
- WM 상담 지원 에이전트
- 트레이딩 데스크 SOP 검색
- 사내 업무 매뉴얼 검색

### 생성 방법
1. 앱 관리 > "+ 새 앱 만들기" > "에이전트 만들기" 선택
2. 앱 정보 (이름/아이콘/설명) 입력
3. LLM 모델 선택 및 인스트럭션 작성
4. 도구 탭에서 MCP 도구 추가
5. 미리보기 테스트 → 버전 저장 → 공개

> 상세 설정은 [에이전트형 앱 가이드](agent-app.md)를 참고하세요.

## 3. 답변형 앱 (Single Action App)

### 개요
단일 입력 → 단일 출력의 간단한 구조입니다. 특정 작업(보고서 생성, 번역, 요약 등)을 반복적으로 수행할 때 적합합니다.

### 주요 특징
- **구조화된 입력**: 텍스트, 문서 첨부, 선택형 등 최대 6개 입력 항목
- **프롬프트 설정**: System/User 프롬프트로 출력 형식 제어
- **Markdown 출력**: 구조화된 출력 지원
- **간편한 사용**: 양식 작성 → 실행 → 결과 확인

### 사용 사례
- 투자제안서(Pitch Book) 초안 생성
- 리서치 보고서 요약
- 문서 번역

### 생성 방법
1. 앱 관리 > "+ 새 앱 만들기" > "답변형 앱" 선택
2. 입력 항목 설정 (문서/텍스트/선택 등)
3. System/User 프롬프트 작성
4. 출력 형식 설정 (Markdown 활성화 등)
5. 테스트 → 공개

> 상세 설정은 [답변형 앱 가이드](answer-app.md)를 참고하세요.

## 공통 기능

### 앱별 접근 권한
- 앱 설정에서 접근 가능한 멤버/그룹을 지정
- "전체", "특정 그룹", "특정 멤버" 중 선택

### 추천 질문 관리
- 앱별로 사용자에게 표시할 추천 질문을 설정
- 사용자가 클릭하면 입력란에 자동 채워짐

### 앱 공개 승인
- 승인자를 지정하여 결재 후 공개되도록 설정 가능
- 민감한 앱의 무분별한 공개 방지
