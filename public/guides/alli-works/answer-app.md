# 답변형 앱 가이드

답변형 앱(Single Action App)은 단일 입력 → 단일 출력 구조로, 특정 작업을 반복적으로 수행할 때 적합합니다.

---

## 1. 답변형 앱 생성

**경로**: Alli 대시보드 > 앱 관리 > + 새 앱 만들기 > 답변형 앱

### 기본 정보
| 항목 | 설명 | 예시 |
|------|------|------|
| **앱 이름** | 앱 표시 이름 | "Pitch Book 초안 생성기" |
| **설명** | 앱 용도 설명 | "투자제안서 초안을 자동 생성합니다" |
| **카테고리** | 앱 분류 | IB |

---

## 2. 입력 설정

사용자가 입력할 항목을 최대 6개까지 설정합니다.

### 입력 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **Paragraph text** | 여러 줄 텍스트 | 대상 기업/산업 설명 |
| **Short text** | 한 줄 텍스트 | 기업명, 금액 |
| **Single select** | 단일 선택 | 제안서 유형: M&A/IPO/유상증자 |
| **Multi select** | 복수 선택 | 포함할 섹션 선택 |
| **Document** | 문서 첨부 | 참고 자료 PDF |
| **Number** | 숫자 | 목표 금액 |

### 입력 설정 예시 (Pitch Book)

| 순서 | 항목명 | 유형 | 설명 |
|------|--------|------|------|
| 1 | 참고 문서 | Document | 지식베이스 문서 첨부 가능 |
| 2 | 대상 기업/산업 | Paragraph text | 분석 대상 정보 |
| 3 | 제안서 유형 | Single select | M&A, IPO, 유상증자, 회사채 발행 |

---

## 3. 프롬프트 설정

### LLM 모델 선택
입력 설정 아래에서 사용할 LLM 모델을 선택합니다.

### System 프롬프트
에이전트의 역할과 제약사항을 정의합니다.

```
당신은 하나증권 IB본부의 투자제안서 작성 전문가입니다. 
첨부된 참고 문서와 입력 정보를 기반으로 Pitch Book 초안을 
구조화하여 작성합니다.

규칙:
- 모든 수치는 출처를 명시합니다
- 가정이 포함된 부분은 [가정] 태그를 붙입니다
- 표와 차트 형태로 데이터를 정리합니다
```

### User 프롬프트
실제 작업 지시사항을 정의합니다. 입력 변수를 `{변수명}`으로 참조합니다.

```
다음 정보로 {제안서_유형} Pitch Book 초안을 작성해주세요:

대상: {대상기업}
참고자료: {참고문서}

구조:
1) Executive Summary
2) 산업 개요
3) 거래 사례 (유사 딜 3건 이상)
4) 밸류에이션 (멀티플 분석)
5) 리스크 요인
6) 실행 일정(안)
```

---

## 4. 출력 설정

| 항목 | 설명 |
|------|------|
| **Markdown 출력** | 구조화된 마크다운 형식 활성화 |
| **출력 길이** | 최대 토큰 수 설정 |

---

## 5. 테스트 및 공개

### 테스트
1. 각 입력 항목에 샘플값 입력
2. "실행" 클릭
3. 출력 결과 검토
4. 프롬프트 조정 → 재실행 → 반복

### 공개
1. 테스트 완료 후 "공개" 클릭
2. 접근 권한 설정 (전체 / 특정 그룹)
3. 카테고리 지정
4. 추천 질문 설정 (선택사항)

---

## 6. 사용 패턴 예시

### 패턴 1: 보고서 생성
- 입력: 주제 + 참고 문서
- 출력: 구조화된 보고서 초안

### 패턴 2: 번역
- 입력: 원문 텍스트 + 대상 언어
- 출력: 번역된 텍스트

### 패턴 3: 요약
- 입력: 원본 문서
- 출력: 핵심 요약 + 키포인트

### 패턴 4: 데이터 분석
- 입력: 엑셀 파일 + 분석 관점
- 출력: 분석 결과 및 인사이트
